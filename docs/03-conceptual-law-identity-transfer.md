# Tier 1.1 — Conceptual Law / Identity Transfer

## Goal
Make conceptual grafts feel like **law / identity transfer**, not mostly like themed zone effects.

## Current gap
Current conceptual grafts are readable and distinct, but they still mostly behave like curated area powers.
They do not yet strongly communicate that a target is temporarily being treated as if it belongs to another concept or law.

## Scope
- Rework existing conceptual grafts so they apply clearer localized rule/identity substitution.
- Keep them temporary, anchored, reversible, and readable.
- Favor a few strong examples over broad expansion.
- Preserve the current conceptual entry flow and command surface.

## Non-goals
- No global world rewriting.
- No freeform concept engine.
- No new progression systems.
- No large architecture rewrite.
- No broad new conceptual graft list in this phase.

## Workflow requirements
- Use a proper git worktree.
- Keep the worktree clean.
- Do not add code comments.
- Keep changes narrow and reviewable.

## Acceptance criteria
- Conceptual grafts read as law/identity transfer, not just zone buffs.
- At least one zone-based conceptual graft and one non-zone conceptual graft clearly behave as rule substitution.
- The concept menu feels more meaningful without adding more clutter.
- Build passes.
- Relevant tests pass.
- Worktree is clean at handoff.
